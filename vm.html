UX/UI CASE STUDY:
DESIGNING A MORE PERSONAL VOLUNTEER OPPORTUNITY FINDING EXPERIENCE FOR VOLUNTEERMATCH.ORG

MY ROLE :

I worked on this project with 2 teammates. We collaborated with strategizing and research. My individual roles with this project were visual style design and high fidelity prototyping.

IN SHORT :

VolunteerMatch is a website that is used for the purpose of pairing people with volunteer opportunities that fit them. Through analysis of the current site in addition to research involving user interviews, we observed that VolunteerMatch was not connecting users to fitting opportunities in an efficient enough manner. We set out to design a solution that addresses this problem. We also set out to improve feedback within the process by which users connect with and maintain contact with volunteer organizations. RESEARCH

CURRENT SITE ANALYSIS :

Research began by taking a look at the current site and finding areas that could be improved to better help users to find volunteer opportunities that fit them. The current site’s landing page is taken up primarily by information about the the organization Volunteer Match, itself. This page can serve the user better instead by presenting him or her with relevant opportunities from the get go. In addition to this, apart from looking dated, the site design is inconsistent across different parts of the site. This results in a disjointed feeling when browsing. Other specific areas of improvement are pointed out under each image below.

1. LANDING PAGE

The landing page presents the user with a hero image with the call to action being a search by location. This can be improved by giving the user more search options. The implementation of an advanced search option was thought of at this point. After the hero image, the rest of the page consists of information about Volunteer Match. This space can be better utilized to help users in finding relevant opportunities.

2. SEARCH RESULTS PAGE

The search results page design is strangely left justified. The space on the right is not utilized at all. Overall, the page looks dull. It can be improved by implementing a better layout design in addition to some colorful images to add visual interest and excitement. The use of a card style system to display results could be a nice solution for this.

3. OPPORTUNITY DETAIL PAGE

The opportunity detail page is one example of inconsistency in the site's design language. It switches to this off white color for no apparent reason. The page layout is awkward with information scattered around seemingly haphazardly. This page can be improved by organizing the information in a more thoughtful and aesthetically pleasing manner.

4. USER PROFILE

This page also suffers from the off white color design switch-up. In addition to this, it uses a tabbed navigation to switch between sections. In my opinion, this looks dated. This can be improved with a more modern design.

USER INTERVIEWS :

Our group interviewed several users of the site. From these interviews we were able to take away several pain points that users encounter when using the site. Here are some standout quotes that helped guide the direction of the re-design.

“It is never acknowledged that my application was received, reviewed, or even appreciated.”
“I don’t trust the site to do a good job of matching me. The profile builder looks old and outdated.”
“The search functionality is lacking a little bit. So I scroll a lot.”

TAKEAWAYS :

From our site analysis and user interviews, we were able to isolate the specific pain-points which we would adress in the re-design. Some observations that were made during the current site analysis were confirmed by conducting the user interviews. These observations include the dated visual design as well as the inefficient search system. Our main takeaways from the research can be summarized below. These would be the key issues that the re-design would set out to tackle.

1. Searching and Sifting

Users spend too much time sorting through search results (an average of 700+ results per search) resulting in increased drop-off rates

2. Irrelevant Results

Results that do no connect with the user are seen as incorrect and do not instill confidence in the service.

3. Trying to Connect

Users report problems when trying to connect with the organizers. A lot of times there is no feedback when an application is submitted, resulting in distrust.

PROPOSED SOLUTION :

The re-design will address the mentioned pain points in the following ways:
  • Users will view fitting opportunities in a visually interesting way as soon as they land on the site.
  • The search/match results will be accurate and relevant.
  • Feedback given to the user in the process of connecting with volunteer organizers will be improved.
  • UI design will be updated to a consistent, clean, and modern look.

THE DESIGN PROCESS

PAPER SKETCHES

The design process started with rough wireframe sketches. I laid out the basic layout and flow of the design. Major design decisions are detailed under each image below.

A: Landing Page

This is the initial concept for the landing page. It maintains the search call to action but replaces the majority of the original content with a card style system to display relevant opportunities to the user. These cards would be a major part of the site’s re-design. The information regarding Volunteer Match would be consolidated to a video at the end of the page.

B: Advanced Search

This is the first version of an advanced search implementation. The idea was that the user would press a button which would reveal all of the advanced search options via an accordion style menu. This would later be revised in favor of a more clean and focused modal implementation.

C: Opportunity Detail Page

We wanted to give the opportunity detail page a more structured layout. This layout was designed to present all of the important information regarding the event in an organized manner which was easy to follow.

D: User Profile Page

A major feature of the re-design was improving profile personalization. The thought behind improving this would be that it would allow users to match with opportunities tailored to their interests, skills, in addition to location. A resume drag and drop feature was also added.

LO-FI WIREFRAMES

After the basic layout of the design was decided upon, we made these wireframes. Some changes were made between these two design stages. The advanced search was changed to a modal that was triggered by a button press on the landing page. This design felt cleaner and focused since the visual distractions of the rest of the page would be darkened and blurred in the background. At this stage, we also really fine-tuned how users would interact with their profile page. As a result, we designed a page in which users would edit their profile with their relevant information. Additonally, a message modal was designed with the purpose of giving the user an opportunity to provide more information specific to the opportunity they were inquiring about. The thought behind this was that it would increase chances of a response from volunteer organizers.

STYLE GUIDE

I created a style guide in order to really nail down a visual style. I kept the colors of the existing site while updating certain UI elements and fonts to give them a more clean and modern look. I wanted the site to look friendly which is why I favored rounded corners on most of the UI elements as well as bright, cheery images. I used clean, flat colors with simple, yet elegant UI elements to give the existing pages a needed facelift. I ended up going with some different styles on the final prototype, but this guide really helped to guide the overall look and feel of the site.
